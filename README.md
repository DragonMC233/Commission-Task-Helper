# 委托排单助手 Task Helper

一个面向画画人的的本地任务管理工具，支持任务卡片管理、甘特图视图、链式任务调度和收入统计，通过浏览器访问，数据存储在本地 JSON 文件中（后续可能会更改为使用sql存储）。

---

## ✨ 主要功能

- **任务管理**：创建、编辑、删除委托任务，记录类别、来源平台、预计工时、稿酬、节点进度、图片附件等信息
- **甘特图视图（BarView）**：可视化排单时间线，支持链式任务自动调度
- **链式任务（线性排单）**：绑定前序任务，自动计算开始/截止日期，可设置缓冲天数
- **收入统计**：按月统计预期/实际收入，目前跨月任务量均摊，收入以完成任务或者增加收款记录的方式计算
- **搜索与过滤**：按任务名称搜索，不过有甘特图之后用处不大
- **回收站**：防手残功能，删除的任务和图片保留 30 天后自动清除
- **其他设备从局域网访问**：启动后手机/平板可通过局域网 IP 访问

## 📁 文件结构

```
task-helper/
├── MainView/               # 任务卡片模式页面
├── BarView/                # 甘特图模式页面
├── scripts/                # 两个网页共用的 JS 模块
├── font/                   # 用于离线运行的本地字体文件（来自Google Material Icons/Symbols）
├── pic/                    # 存放任务图片
├── RecyclePic/             # 存放位于回收站的任务的图片
├── index.json              # 存储任务数据、自定义类型、平台、节点
├── recyclebin.json         # 存放已删除任务的地方
├── Statistics.json         # 任务统计数据
├── basetype.json           # 默认类型/平台/节点预设
├── StartServer.py          # Flask 后端服务器启动脚本
├── package.json            # 前端构建工具依赖（Tailwind CSS）
└── .gitignore
```


 为了防止意外，**请务必定期备份`index.json`文件和`./pic`文件夹**  
如何清空数据：删除 `index.json`并重启StartServer.py，服务器重启后会根据 `basetype.json` 自动生成新的初始数据。

## 🚀 快速开始

### 环境要求

- Python 3.8+
- 依赖库：`flask`、`Pillow`

### 安装依赖

```bash
pip install flask Pillow
```

### 启动服务器

```bash
python StartServer.py
```

启动后访问：
- **主页面**：[http://127.0.0.1:2233](http://127.0.0.1:2233)
- **甘特图**：[http://127.0.0.1:2233/BarView/bar.html](http://127.0.0.1:2233/BarView/bar.html)
- **局域网手机访问**：控制台会显示局域网地址（如 `192.168.x.x:2233`）使用此地址访问即可，甘特图亦支持手机竖屏访问，但横屏使用更佳



## 🎨 自定义

- **网页图标**：将你想要的图标转换为ico文件，并重命名为 `favicon.ico`，放入根目录即可
- **主题样式**：编辑 `MainView/标准&暖色.css` 和 `BarView/bar-standartd&warm.css`
- **默认预设**：编辑 `basetype.json` 修改默认任务类型、平台、节点

- **主题文件位置**：大部分颜色样式位于 `MainView/暖色.css`（暖色主题），你也可以编辑 `MainView/标准.css` 来切换或自定义主题。
